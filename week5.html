Week 5: Accessibility Activities

Activity 1: Spot the Barriers

  • Issue: Low contrast between light grey text and white background
    Who is affected: Users with visual impairments
    Fix: Use dark text on light background with at least 4.5:1 contrast ratio
  • Issue: Error messages shown only in red
    Who is affected: Colour-blind users
    Fix: Add text labels/icons and use ARIA live regions for screen readers
  • Issue: Image CAPTCHA without audio
    Who is affected: Screen reader users
    Fix: Provide audio CAPTCHA or text-based alternatives

Activity 3: Mobile App Analysis

  • Login Page Issue: Tiny buttons, no keyboard focus, Face ID only
    Fix: Increase button size, add focus indicators, provide password option
  • Account Overview Issue: Only pie chart, red/green used
    Fix: Provide text/table summary, use textures/icons for categories
  • Money Transfer Issue: Timeout, disappearing errors, swipe-only
    Fix: Extend timeout, persist errors, provide confirm button

Extension Challenge: Accessibility Guidelines

  1. Use high-contrast colours for text readability
  2. Implement semantic HTML and ARIA roles
  3. Ensure support for keyboard and assistive tech navigation
  4. Avoid motion, auto-scrolling, and flashing content
  5. Include captions/transcripts for all media

Reflection

Accessibility is neglected by many of the digital tools that I utilise. I've come across stuff that is difficult to read and lacking alt text. During the design process, I would recommend including keyboard testing and contrast tests in order to make the design more inclusive..
